## Google Calendar Pipeline

```
Google Calendar dataset
        ↓
Transform 1: normalize timestamps
        ↓
Transform 2: derive date + time fields
        ↓
Transform 3: compute duration / metadata
        ↓
Write to Ontology (TimeBlock)
```

I decided to add a field for day of week and for event length 

There was also a major  issue with timezeone: essentially when I added the data it was already with an offset. My options were to pick EST which would offset it again, or UTC which would get rid of the offset and require a dynamic offset. 

## Final Googlle Calender Pipeline (Writes to Ontology) 
```
Reads Google Calendar events, removes unnecessary details, and adjusts date and time formats. It calculates the duration of events, categorizes them based on calendars, and filters out lengthy events. Additionally, it identifies gaps between scheduled events while ensuring proper time formatting. Finally, it creates new entries for free time slots and consolidates them with the original event data for a comprehensive view of calendar availability.
```
